Why is the JavaScript textContent property safer than innerHTML?

The textContent property is safer than innerHTML. if you use innerHTML, it can input anything. However,textContent treated strictly as text, it makes safer for input.

Explain the relationship between an HTML attribute and a corresponding DOM property (explain the connection to JavaScript). Use a specific example in your answer.

In HTML, tags can have attributes. When the browser parses the HTML and creates DOM objects for the tags, it recognizes the standard attributes and generates DOM properties for them.

<body id="test" something="non-standard">
            <script>
                  alert(document.body.id); // test
                  // the properties will not be generated.
                  alert(document.body.something); // undefined
            </script>
        </body>

Standard attributes are described in the specification of the corresponding element class.
